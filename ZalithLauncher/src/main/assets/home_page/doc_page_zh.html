自定义主页使用指南

欢迎使用 Zalith Launcher 2 的自定义主页！现在主页使用 HTML + JavaScript + CSS 编写，你可以利用完整的 Web 技术栈来构建富交互的主页。

主页文件位于启动器外部存储的 home_page.html，可使用相对路径引用同目录下的 .js / .css / 图片资源。

基本结构

一份最简的主页 HTML：

<!DOCTYPE html>
<html lang="zh">
<head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <style>
        body { font-family: sans-serif; padding: 16px; }
    </style>
</head>
<body>
    <h1>Hello Zalith!</h1>
    <script src="main.js"></script>
</body>
</html>

JavaScript 桥接

启动器向 window.ZalithLauncher 注入了以下方法，供主页与原生层通信：

方法 说明
sendEvent(key, data?) 向启动器发送事件，key 为事件名，data 为可选字符串参数
isDarkTheme() 返回当前是否为深色主题，便于 JS 主动判断主题
getLanguage() 返回当前语言标签（如 zh、en）
建议在调用前做一次存在性检查：if (window.ZalithLauncher) { ... }，方便在普通浏览器中预览调试主页。

支持的事件

下列事件可由 ZalithLauncher.sendEvent(key, data) 触发：

key data 说明
url 完整的 http(s) 链接 在系统浏览器中打开链接
check_update — 触发启动器检查更新
launch_game server=地址（可选） 启动当前选中的游戏版本，可附带快速加入服务器
copy 待复制的文本 将文本写入系统剪贴板
refresh_page — 刷新当前主页
share_game_log — 分享当前游戏版本的最新日志

主题适配

启动器会在页面加载完成时，根据当前主题在 <html> 上添加 dark 或 light 类。建议在 CSS 中通过 CSS 变量切换配色：

:root {
    --bg: #ffffff;
    --text: #1a1a1a;
}
html.dark {
    --bg: #1c1b1f;
    --text: #e6e1e5;
}
body {
    background: var(--bg);
    color: var(--text);
}

当用户切换主题时，启动器会运行时更新 <html> 上的类名，无需重新加载页面。

外部资源

主页 HTML 可通过相对路径引用同目录下的资源：

<link rel="stylesheet" href="style.css">
<script src="main.js"></script>
<img src="banner.png">

也支持绝对 URL 引用网络资源（如 CDN 上的库），但请留意网络可用性与隐私风险。

从网络加载主页

当主页类型为「从网络加载」时，启动器会定期拉取并缓存你填写的 URL 返回的 HTML。同样可以使用相对路径引用同目录下随缓存一起保存的资源。

安全提示

自定义主页是第三方内容，启动器不对任何主页内容负责。请谨慎接受陌生来源的主页文件或主页 URL，避免执行可能涉及账号、隐私或财产风险的脚本。

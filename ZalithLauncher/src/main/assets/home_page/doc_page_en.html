Custom Homepage User Guide

Welcome to Zalith Launcher 2's custom homepage! The homepage is now written with HTML + JavaScript + CSS, so you can use the full web stack to build rich interactive pages.

The homepage file is stored at home_page.html in the launcher's external storage. You can reference .js / .css / image assets in the same directory via relative paths.

Basic Structure

A minimal homepage HTML:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <style>
        body { font-family: sans-serif; padding: 16px; }
    </style>
</head>
<body>
    <h1>Hello Zalith!</h1>
    <script src="main.js"></script>
</body>
</html>

JavaScript Bridge

The launcher injects the following methods into window.ZalithLauncher for the homepage to communicate with the native layer:

Method Description
sendEvent(key, data?) Send an event to the launcher. key is the event name, data is an optional string argument.
isDarkTheme() Returns whether the dark theme is currently active.
getLanguage() Returns the current language tag (e.g. zh, en).
It's recommended to guard the call with an existence check: if (window.ZalithLauncher) { ... }, so you can preview the homepage in a regular browser during development.

Supported Events

The following events can be triggered via ZalithLauncher.sendEvent(key, data):

key data Description
url A full http(s) URL Open the URL in the system browser
check_update — Trigger a launcher update check
launch_game server=address (optional) Launch the currently selected game version, optionally with a server to quick-join
copy The text to copy Copy the given text to the system clipboard
refresh_page — Refresh the current homepage
share_game_log — Share the latest log of the current game version

Theme Adaptation

When the page finishes loading, the launcher adds a dark or light class to <html> based on the current theme. It's recommended to switch colors via CSS variables:

:root {
    --bg: #ffffff;
    --text: #1a1a1a;
}
html.dark {
    --bg: #1c1b1f;
    --text: #e6e1e5;
}
body {
    background: var(--bg);
    color: var(--text);
}

When the user switches theme, the launcher updates the class on <html> at runtime, without reloading the page.

External Resources

The homepage HTML can reference resources in the same directory via relative paths:

<link rel="stylesheet" href="style.css">
<script src="main.js"></script>
<img src="banner.png">

Absolute URLs (e.g. libraries on a CDN) are also supported, but be aware of network availability and privacy risks.

Loading from URL

When the homepage type is set to "Load from URL", the launcher periodically fetches and caches the HTML returned by your configured URL. Relative paths resolve to resources stored alongside the cached HTML.

Security Notice

Custom homepages are third-party content; the launcher takes no responsibility for any homepage content. Be cautious when accepting homepage files or URLs from unfamiliar sources, and avoid running scripts that may compromise your account, privacy, or property.
